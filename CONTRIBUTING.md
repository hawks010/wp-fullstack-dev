# Contributing

Contributions are welcome, but maintenance is automation-first and best-effort. Opening an issue or pull request does not create a support commitment or response-time guarantee.

## Before contributing

1. Use the structured issue forms. Blank issues are disabled.
2. Do not disclose vulnerabilities publicly; follow [SECURITY.md](SECURITY.md).
3. Keep each pull request focused on one outcome.
4. Do not add client branding, secrets, generated dependency folders, or unrelated formatting changes.

## Pull requests

Fork the repository, create a focused branch, and include evidence for the checks you actually ran. The automated queue labels changed areas, welcomes first-time contributors, checks the examples, and eventually closes abandoned work.

Required before merge:

- CI passes.
- New behavior has proportionate tests.
- Documentation and changelog are updated when user-facing behavior changes.
- Security, accessibility, and compatibility claims are evidence-based.
- Compiled production assets are committed where the distributed starter requires them.

Pull requests remain subject to maintainer approval. AI and repository automation never approve or merge untrusted contributor code.

## Forks

Public forks are independent repositories controlled by their owners. They are not maintained, endorsed, monitored, or supported by Sonny x Inkfire unless their changes are submitted here as a pull request.
