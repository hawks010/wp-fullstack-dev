# Support

This is an open-source developer toolkit, not a managed support service.

- Use GitHub Discussions for usage questions and ideas.
- Use structured Issues for reproducible defects or scoped feature proposals.
- Use private vulnerability reporting for security concerns.
- Client-site debugging, deployment, and production incidents are outside community support scope.

Automated triage may classify, label, or close inactive threads. A bot response does not mean a maintainer has accepted the request.
