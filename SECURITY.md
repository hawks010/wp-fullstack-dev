# Security policy

## Supported version

Security fixes target the latest tagged release.

## Reporting a vulnerability

Do not open a public issue containing exploit details, credentials, personal data, or a working proof of concept. Use GitHub's private vulnerability reporting for this repository when available.

Include the affected file or component, impact, reproduction conditions, and a suggested mitigation if known. Reports are reviewed on a best-effort basis. Do not test against systems or data you do not own or have explicit permission to assess.

AI triage is never used to publish, reproduce, or automatically fix confidential vulnerability details.
